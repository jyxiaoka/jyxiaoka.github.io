      iOS开发-第三方Masonry实现AutoLayout布局

      jyxiaoka 2016-10-23 iOS

      当我们使用苹果官方的自动布局时，会感觉到非常的繁琐。使用代码实现autolayout是往往需要写一堆又臭又长的代码，不会给人通俗易懂的感觉，即使使用IB来实现AutoLayout布局也是非常繁琐的一件事情。而第三方自动布局工具——Masonry是一个语法更简洁，代码更优美的轻量级的AutoLayout框架。

      # Masonry介绍

      Masonry是一个轻量级的自动布局框架，拥有自己的描述语法，采用链式语法封装了自动布局，语法简洁，可读性好，同时还支持iOS和MacOS。Masonry的下载可从github上或者使用CocoaPods来安装,可以参考我的另一篇文章iOS开发-使用CocoaPods做依赖管理。 Masonry源码：https://github.com/Masonry/Masonry 包含了Masonry的demo。 上一篇关于AutoLayout的文章说过，AutoLayout的核心就是参照和约束，而Masonry的添加约束的思维跟苹果的原API是相同的，其实只是对原有的API进行了封装，是的添加约束变得简洁明了，添加约束的代码更加优雅了。

      # Masonry的使用

      首先将通过GitHub下载到的Masonry拷贝到项目中，或者通过CocoaPods的pod Masonry将它添加到项目中，然后的要使用约束的viewController中#import "Masonry.h"，就可以使用Masonry了。下面是一个简单的例子：

      UIView *view = [[UIView alloc] init];
view.backgroundColor = [UIColor greenColor];
//需先add到父view中
[self.view addSubview:view];
//调用Masonry的UIView的分类方法，在block中实现约束的添加
[view mas_makeConstraints:^(MASConstraintMaker *maker){
	//view的中心=父view的中心
	maker.center.equalTo(self.view);
	//设置view的大小
	maker.size.mas_equalTo(CGSizeMake(200, 180));
}];

      以上是简单的实现一个view在父view中的布局。现在来实现4个view的相对布局，Masonry的语法在上面的例子中也可以看到，它容易使用，代码简洁明了易懂，不用很多代码就可以轻松实现自动布局，而且写起来很顺手。

      	//首先创建4个view并添加到父view中
	UIView *redView = [[UIView alloc] init];
    redView.backgroundColor = [UIColor redColor];
    [self.view addSubview:redView];  
    
    UIView *greenView = [[UIView alloc] init];
    greenView.backgroundColor = [UIColor greenColor];
    [self.view addSubview:greenView];    
    
    UIView *blueView = [[UIView alloc] init];
    blueView.backgroundColor = [UIColor blueColor];
    [self.view addSubview:blueView];
    
    UIView *yellowView = [[UIView alloc] init];
    yellowView.backgroundColor = [UIColor yellowColor];
    [self.view addSubview:yellowView];
    
    //设置红色view距离父视图左边距30px 上边距100px 大小(100，80)
    [redView mas_makeConstraints:^(MASConstraintMaker *make){
        make.left.offset(30).with.top.offset(100);
        make.size.mas_equalTo(CGSizeMake(100, 80));
    }];
    
    //设置绿色view左边距=红view的右边距+20 Y中心店=红view的Y中心点，大小=红view
    [greenView mas_makeConstraints:^(MASConstraintMaker *make){
        make.left.mas_equalTo(redView.mas_right).offset(20);
        make.centerY.mas_equalTo(redView);
        make.size.mas_equalTo(redView);
    }];
    
    //蓝view左边距=红view，上边距=红view的下边距+30，大小=红view
    [blueView mas_makeConstraints:^(MASConstraintMaker *make){
        make.left.mas_equalTo(redView);
        make.top.mas_equalTo(redView.mas_bottom).offset(30);
        make.size.mas_equalTo(redView);
    }];
    
    //黄view左边距=绿view左边距，上边距=绿view上边距+30 大小=绿view
    [yellowView mas_makeConstraints:^(MASConstraintMaker *make){
        make.left.mas_equalTo(greenView);
        make.top.mas_equalTo(greenView.mas_bottom).offset(30);
        make.size.mas_equalTo(greenView);
    }];

      最后呈现的就是四个整齐的方块。这里只是介绍了比较简单的布局使用，更多的是要自己去动手写，才能体会其精髓。而且Masonry是开源的，可以查看它的源代码，学习它是怎么封装的。分析并学习它的思想，有助于我们进步。
